# 功能计划书

## 功能一：AI一键导入频率

### 1. 功能概述
用户可以通过粘贴未经处理的频率文本，由AI自动解析、排序并转换为标准JSON格式，预览后可选择取消、重新生成或确认导入。

### 2. 用户流程设计

```
用户点击"AI导入"按钮
    ↓
弹出AI导入模态框
    ↓
用户粘贴原始频率文本
    ↓
用户点击"解析"按钮
    ↓
显示加载状态（调用AI API）
    ↓
AI返回解析结果
    ↓
显示JSON预览（表格形式，便于阅读）
    ↓
用户选择：
  - 取消：关闭模态框
  - 重新生成：重新调用AI
  - 确认：将频率数据填充到表单
```

### 3. 界面设计

#### 3.1 AI导入按钮位置
- 在管理后台的机场编辑模态框中，频率编辑区域的上方
- 按钮样式：与"添加频率"按钮并列，使用不同颜色区分（如紫色）

#### 3.2 AI导入模态框内容
- **标题**：AI一键导入频率
- **文本输入区**：多行文本框，placeholder提示示例格式
- **解析按钮**：点击后调用AI API
- **预览区域**：解析成功后显示，以表格形式展示：
  | 类型 | 序号 | 前备注 | 频率 | 备用频率 | 后备注 |
- **操作按钮**：取消 / 重新生成 / 确认导入

### 4. AI提示词设计

```
你是一个机场无线电频率数据解析助手。请将用户输入的原始频率文本解析为JSON格式。

## 输入格式说明
用户输入的是未经处理的频率文本，每行一个频率，格式可能包含：
- 类型（D-ATIS/APP/TWR/APN/GND/Delivery）
- 序号（如TWR01中的01）
- 前备注（类型后括号内的内容，如D-ATIS(Chinese)中的Chinese）
- 频率值（数字，单位MHz）
- 备用频率（频率后括号内的纯数字）
- 后备注（频率后括号内的文字说明，如DCL AVBL）

## 排序规则
按以下顺序排列：
1. D-ATIS
2. APP（按序号升序）
3. TWR（按序号升序）
4. APN（按序号升序）
5. GND（按序号升序）
6. Delivery（按序号升序）

## 输出要求
仅输出纯JSON数组，不要包含任何解释文字或markdown代码块标记。

JSON格式：
[
  {
    "type": "类型",
    "number": 序号数字或null,
    "preRemark": "前备注或null",
    "frequency": "频率值",
    "backup": "备用频率或null",
    "postRemark": "后备注或null"
  }
]

## 用户输入：
{{用户输入的文本}}
```

#### 5.1 API设置存储

```javascript
    },
    {

      "content": "用户输入的频率文本"
    }
  ],
  "temperature": 0.1

```

### 6. 设置界面设计

#### 6.1 入口位置
- 管理后台页面右上角添加"设置"按钮（齿轮图标）
- 点击后弹出设置模态框

#### 6.2 设置项
| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| API地址 | AI服务的API端点 | https://api.openai.com/v1/chat/completions |
| API密钥 | 用于认证的密钥 | 空 |
| 模型名称 | 使用的模型 | gpt-3.5-turbo |

#### 6.3 设置验证
- 提供"测试连接"按钮
- 发送简单请求验证API是否可用
- 显示成功/失败提示

### 7. 错误处理

| 错误情况 | 处理方式 |
|----------|----------|
| API密钥未设置 | 提示用户先在设置中配置API |
| API请求失败 | 显示错误信息，建议检查网络或API设置 |
| AI返回格式错误 | 提示"解析失败，请重试或手动输入" |
| JSON解析失败 | 同上 |

## 功能二：前台隐藏管理后台入口

### 1. 功能概述
在首页（频率查询页面）隐藏管理后台的导航链接，用户需要通过特定方式访问管理后台。

### 2. 实现方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 方案A：完全移除链接 | 简单直接 | 用户无法直接发现入口 | ⭐⭐⭐ |
| 方案B：通过URL直接访问 | 隐藏入口但保留访问能力 | 需要记住URL | ⭐⭐⭐⭐⭐ |
| 方案C：双击Logo进入 | 有趣的隐藏方式 | 可能被误触发现 | ⭐⭐⭐⭐ |
| 方案D：组合键进入（如Ctrl+M） | 专业感强 | 移动端无法使用 | ⭐⭐⭐ |

### 3. 推荐方案：方案B + 方案C组合

#### 3.1 主要访问方式
- 直接访问 `/admin.html` URL

#### 3.2 辅助访问方式
- 在首页连续点击标题5次，显示管理后台入口
- 入口显示后5秒自动隐藏

### 4. 实现细节

#### 4.1 导航栏修改
- 移除管理后台的显式链接
- 添加一个隐藏的管理入口（默认display: none）

#### 4.2 彩蛋触发逻辑
```javascript
// 伪代码
点击计数 = 0
上次点击时间 = 0

当标题被点击时：
  如果 当前时间 - 上次点击时间 < 500ms：
    点击计数++
  否则：
    点击计数 = 1
  
  上次点击时间 = 当前时间
  
  如果 点击计数 >= 5：
    显示管理入口
    5秒后隐藏管理入口
    点击计数 = 0
```

### 5. 文件修改清单

| 文件 | 修改内容 |
|------|----------|
| `public/index.html` | 移除管理后台链接，添加隐藏入口 |
| `public/js/app.js` | 添加彩蛋触发逻辑 |
| `public/css/style.css` | 添加隐藏入口样式 |
